# Media Stream Analyzer v2 — Agent Guidelines

## Обязательное правило (CRITICAL)

**При создании новых страниц, секций и элементов ПО обязано руководствоваться требованиями, описанными в `v2/docs/DESIGN_SYSTEM.md`.**

Любое отклонение от Design System требует:
1. Обновления `DESIGN_SYSTEM.md` **перед** изменением кода
2. Утверждения изменений от пользователя

---

## Порядок работы с дизайном

1. **Открыть `v2/docs/DESIGN_SYSTEM.md`** — проверить цвета, типографику, layout, компоненты
2. **Проверить `v2/src/frontend/css/styles.css`** — убедиться, что CSS variables существуют
3. **Создать/изменить HTML** — использовать только утверждённые классы и стили
4. **Обновить `DESIGN_SYSTEM.md`** — если создан новый компонент, задокументировать его

---

## Что нельзя делать без разрешения

- Изменять цветовую палитру (`:root` переменные)
- Изменять типографику (шрифты, размеры)
- Изменять layout grid (sidebar width, header height)
- Добавлять новые цвета вместо существующих
- Использовать gradients на метрах (только solid colors)

## Что можно делать свободно

- Добавлять новые компоненты, если они следуют BEM-нотации и используют CSS variables
- Добавлять новые страницы анализаторов (по шаблону audio/video)
- Добавлять новые протоколы в select
